## apM Claw란?

apM Claw는 크립토 및 Web3 프로젝트를 위한 **24시간 AI 기반 텔레그램 커뮤니티 매니저**입니다.

**헤리티지:**
- **2023년부터**: PM-E가 AI 기반 스팸 탐지로 apM 커뮤니티를 지원
- **2025년**: OpenClaw의 아키텍처 기반, 크립토 보안에 특화하여 재탄생

**왜 "apM"?**
AM + PM = 24시간. 커뮤니티는 잠들지 않고, apM Claw도 마찬가지입니다.

**왜 OpenClaw?**
크립토 커뮤니티는 **전체 컨텍스트 분석**이 필요한 정교한 피싱 공격에 직면합니다. OpenClaw의 아키텍처는 다음을 가능하게 합니다:
- 다중 턴 대화 이해
- 관리자 권한 검증
- 컨텍스트 인식 위협 탐지
- 행동 패턴 분석

**OpenClaw 기반:**
[OpenClaw](https://openclaw.ai)에서 리스펙트를 담아 포크했습니다. 강력한 게이트웨이 아키텍처를 계승하며, 텔레그램 커뮤니티 보안에 집중합니다.

## 주요 기능

### 커뮤니티 관리 (PM-E에서 계승)
- **AI 기반 안티 스팸** - 정교한 스캠을 탐지하는 전체 컨텍스트 분석
- **관리자 권한 제어** - 모더레이션 액션 허용 전 관리자 상태 검증
- **자동 환영** - 커스터마이징 가능한 신규 멤버 환영 메시지
- **스마트 자동 응답** - 프로젝트 문서(백서, 웹사이트, Medium 등)에서 FAQ 답변
- **다국어 지원** - 한글/영어 채널 별도 처리

### OpenClaw 프레임워크 기반
- **텔레그램 + 디스코드만** - 크립토 커뮤니티 플랫폼에 집중
- **컨텍스트 아키텍처 유지** - OpenClaw의 메모리 & 세션 관리 보존
- **라우팅 단순화** - 인스턴스당 단일 에이전트 (복잡한 멀티 에이전트 라우팅 제거)
- **30개 이상 LLM 제공자** - OpenAI, Anthropic, Gemini, 로컬 모델 등
- **영속적 세션** - 채팅방별 대화 컨텍스트 유지

📚 **더 알아보기**: [OpenClaw 개념](https://docs.openclaw.ai/concepts) | [에이전트 시스템](https://docs.openclaw.ai/concepts/agent)

### 크립토 친화적 (Web3 최적화)
- **브라우저 자동화** - 커뮤니티 모니터링, 차트 스크린샷 캡처 (보안을 위해 제한된 범위)
- **웹 스크래핑** - FAQ 응답을 위한 프로젝트 문서 가져오기 (공격 방지 검토 중)
- **API 통합 준비** - CoinMarketCap, CoinGecko 지원
- **포크 친화적** - 다른 프로젝트가 쉽게 커스터마이징 가능

📚 **더 알아보기**: [브라우저 도구](https://docs.openclaw.ai/tools/browser) | [웹 페치](https://docs.openclaw.ai/tools/web-fetch)

## 빠른 시작

### 사전 요구사항
- **Docker & Docker Compose** (필수)
- **Linux** (프로덕션 권장)
- 텔레그램 봇 토큰 ([@BotFather](https://t.me/botfather)에서 발급)
- AI 모델 API 키 (Anthropic, OpenAI 등)

### 설치 (Docker 전용)

apM Claw는 일관되고 안전한 배포를 위해 Docker를 사용합니다.

```bash
# 1. 레포지토리 클론
git clone https://github.com/apmcoin/apmclaw.git
cd apmclaw

# 2. .env 파일 생성
cp .env.example .env
# .env 편집 후 다음 입력:
# - OPENCLAW_GATEWAY_TOKEN (생성: openssl rand -hex 32)
# - TELEGRAM_BOT_TOKEN (@BotFather에서)
# - ANTHROPIC_API_KEY 또는 OPENAI_API_KEY

# 3. Docker Compose로 시작
docker compose up -d

# 4. 로그 확인
docker compose logs -f

# 5. 헬스 체크
curl http://localhost:18789/healthz
```

📚 **왜 Docker 전용?**: 일관된 환경, 더 나은 격리, 쉬운 배포. npm 글로벌 설치나 데몬 설정 불필요.

### 기본 사용법

```bash
# 로그 보기
docker compose logs -f apmclaw

# 서비스 재시작
docker compose restart apmclaw

# 서비스 중지
docker compose down

# 코드 변경 후 재빌드
docker compose build && docker compose up -d

# CLI 접근 (디버깅용)
docker compose --profile cli run apmclaw-cli agent --message "안녕!"
```

## ⚠️ 보안 경고

### AI는 완벽하지 않습니다

**중요**: AI는 속을 수 있습니다. 크립토 커뮤니티의 경우:

1. **AI 인젝션 공격** - 악의적 사용자가 프롬프트 인젝션으로 필터를 우회하려 시도할 수 있음
2. **민감한 시스템 분리** - 다음과 같은 것들과 같은 인스턴스에서 apM Claw를 **절대 실행하지 마세요**:
   - 지갑 개인 키
   - 금융 접근 권한이 있는 API 키
   - 토큰 컨트랙트 관리
   - 중요 인프라
3. **전용 인스턴스 사용** - apM Claw를 격리된 서버에서 실행
4. **로그 모니터링** - 모더레이션 결정을 정기적으로 검토
5. **인간 감독** - AI는 보조 도구이지, 인간 관리자를 대체하지 않음

### DM 정책

**모든 DM은 기본적으로 차단됩니다.** apM Claw는 **그룹 관리자 모드 전용**으로 작동합니다.

이는 다음을 방지합니다:
- 스팸 DM을 통한 토큰 낭비 공격
- 개인 메시지를 통한 피싱 시도
- 봇 기능에 대한 무단 접근

### 도움이 필요하신가요?

보안 설정이 불확실하다면, **apM 팀에 문의**하여 도움을 받으세요.

## 설정

주요 설정 파일:
- `~/.config/apmclaw/openclaw.json` - 메인 설정 (JSON5 형식)
- `~/.config/apmclaw/AGENTS.md` - 에이전트 메모리 및 컨텍스트
- `~/.config/apmclaw/SOUL.md` - 봇 페르소나

📚 **전체 레퍼런스**: [OpenClaw 설정 가이드](https://docs.openclaw.ai/gateway/configuration)

### 예시: 텔레그램 보안 설정

```json5
{
  channels: {
    telegram: {
      enabled: true,
      botToken: "YOUR_BOT_TOKEN",
      dmPolicy: "blocked",  // DM 완전히 비활성화
      allowFrom: ["tg:@YourProjectAdmin"]  // 관리자 화이트리스트
    }
  },
  agents: {
    defaults: {
      model: { primary: "anthropic/claude-opus-4-6" },
      workspace: "~/.config/apmclaw/workspace"
    }
  }
}
```

### FAQ 자동 응답 설정

apM Claw는 프로젝트 문서에서 자동으로 응답을 생성할 수 있습니다:

```bash
# 백서 URL 추가
apmclaw faq add --url https://yourproject.com/whitepaper.pdf

# Medium 아티클 추가
apmclaw faq add --url https://medium.com/@yourproject

# 웹사이트 추가
apmclaw faq add --url https://yourproject.com/docs
```

AI는 사용자 질문에 답변할 때 이 문서들을 참조합니다.

## 텔레그램 설정

1. 봇 생성: [@BotFather](https://t.me/botfather) → `/newbot`
2. 봇 토큰 복사
3. 봇을 그룹에 추가
4. **봇에게 관리자 권한 부여** (모더레이션 및 관리자 검증에 필수)
5. `.env` 파일에 토큰 추가: `TELEGRAM_BOT_TOKEN=your-token-here`
6. `config/openclaw.json` 생성 (선택사항, 커스텀 설정용):
   ```json5
   {
     channels: {
       telegram: {
         enabled: true,
         dmPolicy: "blocked"  // DM 기본 비활성화
       }
     }
   }
   ```
7. 시작: `docker compose up -d`

📚 **상세 설정**: [OpenClaw 텔레그램 가이드](https://docs.openclaw.ai/channels/telegram)

apM Claw는 포크하기 쉽게 설계되었습니다:

```bash
# 1. GitHub에서 포크
# 2. 포크한 레포 클론
git clone https://github.com/your-org/apmclaw.git
cd apmclaw

# 3. 설정 커스터마이징
cp .env.example .env
# .env를 토큰으로 편집
```

## 아키텍처

OpenClaw의 **게이트웨이 + 에이전트** 아키텍처를 계승하되, 크립토 보안을 위해 단순화:

- **게이트웨이** - CLI 전용, 채팅방 집중 (원격 노드 없음)
- **에이전트** - 단일 에이전트가 별도 컨텍스트로 모든 채팅방 관리
- **채널** - 텔레그램 + 디스코드만
- **익스텐션** - 검증된 플러그인만

```
┌─────────────────┐
│  텔레그램 봇     │
│  (관리자 전용)   │
└────────┬────────┘
         │
    ┌────▼─────┐
    │ 게이트웨이 │ ←─── CLI 전용, 원격 노드 없음
    └────┬─────┘
         │
    ┌────▼─────┐
    │ 에이전트  │ ←─── 단일 AI, 다중 방 컨텍스트
    └────┬─────┘
         │
    ┌────▼─────┐
    │  도구    │ ←─── 제한된 범위 (보안)
    └──────────┘
```

📚 **더 알아보기**: [OpenClaw 아키텍처](https://docs.openclaw.ai/concepts/architecture)
